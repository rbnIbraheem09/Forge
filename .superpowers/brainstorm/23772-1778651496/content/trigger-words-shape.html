Trigger words — which shape feels right?

LoRA detail page. You'll have a [Paste] button on all three. Question is about the shape of the field itself + how copy-back-out works.

A

Single-line input

One comma-separated string. Simplest. Whole value copies on click.
Trigger words
1girl, masterpiece, <lora:my-lora:1.0>
Click the field → copies whole string
B

Multi-line textarea

Freeform. Good when a LoRA has paragraphs of guidance, sample prompts, etc.
Trigger words
1girl, masterpiece, <lora:my-lora:1.0>
negative: lowres, bad anatomy
recommended weight: 0.7–0.9
Top-right copy icon → copies whole textarea
C

Chip list

Each trigger word/phrase is a separate chip. Click a chip → that one phrase copies. [Paste] splits the clipboard on commas into multiple chips.
Trigger words
1girl × masterpiece × <lora:my-lora:1.0> × + add
Click any chip → copies just that phrase. Most flexible for re-use.

My lean: B (multi-line textarea) — matches the existing Notes field aesthetic, low-cost, and trigger-word formats vary a lot between LoRAs (sometimes one phrase, sometimes a paragraph with weight hints). Chips are nicer for copying individual phrases but add UI complexity for a feature you'll mostly read.
