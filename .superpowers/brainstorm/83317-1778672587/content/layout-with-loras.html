A few things baked into this mockup — call out anything you'd change:

  • Right panel lists all your LoRAs from the existing loras table. Searchable. Each row shows the LoRA name + trigger words. Click row or checkbox to toggle selection. Selected count shown top-right.
  • Sage-green selected pills appear above the textarea — visual confirmation of what's attached. Click × to deselect.
  • Red dashed tags in the output = LoRA trigger words. The AI is told "these are not Danbooru tags, insert them where they make sense and label them as lora-trigger type in your JSON response."
  • Addendum at the bottom is the .safetensors filename of each LoRA — non-copyable, informational only. Stays with the prompt when saved.
  • LoRAs with no trigger words set appear dimmed but still selectable (some users use LoRAs purely for style without trigger words). We can disable them entirely instead if you'd prefer.
  • Offline LoRAs (file missing) appear dimmed with a small badge.
