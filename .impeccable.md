# Forge — Impeccable Design Context

## Design Context

### Users
Solo AI artists and power users who run ComfyUI generation sessions — often late at night, in dim or dark environments. They are technically capable but creatively driven. They use Forge as a personal creative journal: logging, comparing, and reflecting on generative work. The job: keep a structured record (database thinking) AND feel inspired by what they've made (creative journal thinking).

### Brand Personality
**3 words: structured, curious, alive**

Physical object: a Moleskine dotted notebook with graph-paper inserts — precise grids, but filled with sketches, annotations, crossed-out ideas. Personal, not corporate. Deliberate, not sterile.

Emotional goal: organized and in control + inspired by the work logged + efficient. The app should feel like it respects the creative process.

### Aesthetic Direction

**Theme**: Dark — warm-tinted near-black. Slightly amber/ochre-tinted surfaces. Never cool blue-gray (server-room gloomy). Think tungsten light in a dark studio.

**Primary accent**: Yellow (`#e8c820`) — warm vivid gold-yellow. The color of pencil marks, tungsten bulbs, precision instrument dials. Used for primary actions, active states, starring, CTA buttons.

**Secondary accent**: Blue (`#7aa0e8`) — periwinkle/cornflower. Structural, cool, precise. Used for checkpoints, metadata info, secondary interactive elements.

**Tertiary**: Sage (`#7daa88`) — muted sage green. Used for LoRAs.

**References**:
- Arc browser: distinctive, opinionated, personality-driven
- Notion: clean hierarchy, editorial calm, database structure
- ComfyUI: serious dark tooling, purposeful

**Anti-references**: No side-stripe card borders. Not gloomy/plain/boring. Not AI-purple-on-cool-gray. Not over-rounded bubbly. Not generic dev-tool aesthetic.

### Typography

- **Display/headings**: Bricolage Grotesque (Google Fonts, variable) — optically alive at all sizes, mathematical yet playful
- **Body/UI**: Figtree (Google Fonts) — friendly geometric, excellent tabular numerals for seeds/steps/CFG

Google Fonts import:
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Bricolage+Grotesque:opsz,wght@12..96,300..800&family=Figtree:wght@300..800&display=swap" rel="stylesheet">
```

CSS:
```css
font-family: 'Figtree', sans-serif;          /* body/UI default */
font-family: 'Bricolage Grotesque', sans-serif; /* headings */
```

### Motion System

**Library**: Framer Motion (`framer-motion`)
**Easing**: `[0.16, 1, 0.3, 1]` (ease-out-expo) everywhere

**Animation variants:**
```js
const easeExpo = [0.16, 1, 0.3, 1]

// Page/content entrance
export const fadeUp = {
  hidden: { opacity: 0, y: 8 },
  visible: { opacity: 1, y: 0, transition: { duration: 0.26, ease: easeExpo } }
}

// Stagger container
export const stagger = {
  hidden: {},
  visible: { transition: { staggerChildren: 0.05, delayChildren: 0.05 } }
}

// Overlay scale-in (SearchOverlay, CompareOverlay)
export const scaleIn = {
  hidden: { opacity: 0, scale: 0.96 },
  visible: { opacity: 1, scale: 1, transition: { duration: 0.22, ease: easeExpo } }
}

// Panel slide from right (MetadataPanel)
export const slideInRight = {
  hidden: { opacity: 0, x: 32 },
  visible: { opacity: 1, x: 0, transition: { duration: 0.28, ease: easeExpo } }
}

// Toast slide up
export const toastVariant = {
  hidden: { opacity: 0, y: 16, scale: 0.96 },
  visible: { opacity: 1, y: 0, scale: 1, transition: { duration: 0.22, ease: easeExpo } },
  exit: { opacity: 0, y: 8, scale: 0.95, transition: { duration: 0.18, ease: [0.4, 0, 1, 1] } }
}
```

**Key micro-interactions:**
- Star toggle: scale burst 1 → 1.35 → 1 on click (150ms), fills yellow
- Gallery thumbnail hover: scale(1.03) + brightness(1.08), action icons fade in
- Sidebar nav active: background slides/fades in
- Card hover: subtle translateY(-1px) + border brightens
- Assign panel (Inbox): slides up from bottom (AnimatePresence)
- MetadataPanel: slides in from right (AnimatePresence)
- SearchOverlay: scaleIn backdrop fade (AnimatePresence)
- Dashboard stats: useSpring count-up on mount
- Bar chart bars: width 0 → final (staggered, 0.4s duration)
- Toast: slide up → hold → slide down (AnimatePresence)
- Page route change: fadeUp with AnimatePresence in App.jsx

### Design Principles

1. **Warm dark, never cool dark.** Every surface tints toward amber-ochre.
2. **Images are the hero.** Chrome recedes; generated images are what the app is about.
3. **Numbers as design elements.** Seeds, CFG, iteration numbers treated typographically with tabular numerals.
4. **Precision with personality.** Spacing varied for hierarchy, never uniform. Interactions feel alive.
5. **Motion serves meaning.** Every animation communicates state change — nothing decorative.
